Interface LibraryHarness

One library's implementation of all seven scenarios. Implementations live alongside their library imports in src/libraries/<name>.ts.

run returns a RunResult discriminated record (not a single number) so the bench can surface glitches/notifications/recomputes separately — see PR #200 review comments (cross-cutting #9).

computeEndState

  • computeEndState(scenario: ScenarioName, scale: ScenarioScale): unknown

    Run the scenario once at the given scale and return the workload's terminal state (the value the subscriber/observer would see at the very last commit). Used by the conformance suite to assert cross-library equality of the canonical workloads — see @causl/bench/testing (expectedEndState).

    The returned shape is per-scenario (a single number for linear-chain / diamond / dynamic-dep-flip / async-race / equality-cutoff; an array of numbers for scrolling-viewport / batch-commit). The contract is "deep-equals expectedEndState(scenario, scale)."
