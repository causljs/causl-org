Module @causl/bench

@causl/bench

Comparative benchmarks: Causl vs Jotai vs Redux Toolkit vs MobX across the canonical seven-workload scenario contract (#167).

Status

The harness has shipped. Methodology and headline charts live in docs/benchmark.md; this package is the implementation.

  • Library harnesses (src/libraries/): causl.ts, jotai.ts, redux.ts, mobx.ts, plus _instrumentation.ts for the shared per-cell counter shape (glitches / notifications / recomputes).
  • In-process layer (src/run.ts + src/scenario.ts): runs every (library × scenario) cell against tinybench in the same Node.js process. 200ms warm-up, 1000-iteration sample, median reported, noise floor recorded.
  • Playwright DOM layer (e2e/): renders the same scenarios in a production build under Chromium, counts dropped frames at 60 Hz using the requestAnimationFrame delta technique (docs/benchmark.md §"Methodology Notes").
  • Determinism + counter-baseline gate (test/runAll.test.ts): every (library × canonical scenario) cell at scale=100 runs twice and the integer counters must match (SPEC §15.2 replay determinism), then each cell's counters are compared against fixtures/baseline.json exactly. Wall time is checked as a soft 3× ceiling against the baseline median for cells whose baseline median exceeds 1 ms.
  • Baseline regenerator (test/regen-baseline.ts): rerun when an intentional engine change moves the counters; commit fixtures/baseline.json in the same PR.
  • Python launcher (tools/bench/run-bench.py): pinned-Docker reproducer with a typed exit-code contract — 0 success, 10 config error, 11 bench harness failed, 12 JSON parse failed, 13 output write failed, 14 Docker invocation failed, 20 environment guard failed (full table in docs/benchmark.md §"Reproducer exit codes"). The launcher deliberately never propagates the child's raw return code so CI scripts can branch on failure class without pattern-matching log lines.
  • Bundle-size table renderer (src/bundle-table.ts, tools/bench/render-bundle-table.ts): splices a fresh size-limit --json table between markers in docs/benchmark.md on every nightly bench run, so the bundle numbers in the article cannot drift from the size-limit gate.
  • Article-figures generator (src/article-figures.ts + src/chart.ts): produces the SVG charts referenced from docs/benchmark.md §"The Numbers".

Refresh procedure when an intentional engine change moves counters:

pnpm --filter @causl/bench exec tsx test/regen-baseline.ts
git add packages/bench/fixtures/baseline.json

The seven workloads

Scenario Description Size
linear-chain 1000-node chain; bump head; observe tail 1000
diamond A→{B,C}→D; bump A; glitch-free recomputation 4
dynamic-dep-flip Conditionally read A or B; abandoned input does not retrigger 3
scrolling-viewport 1000-cell virtualized grid under scroll 1000
async-race Late fetch must not overwrite newer state 1
batch-commit 50 inputs written in one commit; observers fire once per commit 50
equality-cutoff Same-value recompute does not propagate downstream 10

Same operations, same dataset sizes, same observation points across every library. Tinybench measures wall-clock; Playwright measures dropped frames.
