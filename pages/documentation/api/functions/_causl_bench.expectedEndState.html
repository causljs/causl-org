Function expectedEndState

  • expectedEndState(scenario: ScenarioName, scale: ScenarioScale): unknown

    Pure, deterministic terminal value for a canonical workload at a given scale. Falsifies a regression that silently no-ops the workload (drops the dispatch, swaps set(a, 1) for set(a, 0), returns the wrong tail, etc.) — none of those would survive expect(harness.computeEndState(s, scale)).toEqual(expectedEndState(s, scale)).

    Per scenario, the terminal value is the value the subscriber/observer would see at the very last commit:

    • linear-chain: tail of head→c0→c1→…→cN-1 after set(head, 1). Each cell is prev + 1, head = 1, tail = 1 + scale.
    • diamond: D = b + c = (a+1) + (a*2) = 3a + 1; after set(a, 5), D = 16.
    • dynamic-dep-flip: after set(flag, true) then a write to the abandoned a input (99), D reads b (= 2). If the engine fails to abandon a, D would re-pick it up and not equal 2.
    • scrolling-viewport: cell at index i has value i for every bumped index. Reported terminal is the array of cell values (zero where untouched).
    • async-race: last write wins, x = 2.
    • batch-commit: every input[i] = i + 1 inside one batched commit. Reported terminal is the array of input values.
    • equality-cutoff: downstream derivation = 42 (constant); cutoff means downstream subscribers don't fire on identity-stable recompute.

    Parameters

    • scenario: ScenarioName
    • scale: ScenarioScale

    Returns unknown
